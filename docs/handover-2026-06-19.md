## What's PROVEN (don't re-litigate)
- **Strength ladder** (cref_mos vs SF-1800): d4 1753 / d5 1850 / d6 1942 / **d7 2013**.
- **Hardware**: Ultimate 64 = **64 MHz** (not 40), Nova = 12 MHz. @64MHz: d6=19s,
  d7=67s/move. **1900+ is achieved on the Ultimate, engine as-is.** Cyc=cycle-exact
  sim = real HW.
- **Colossus**: ~1700 / ~20M cyc/move. We beat it +250-300 Elo at depth. The 20M was
  the wrong target (it's a 1MHz number; we target fast HW).
- **Forward pruning is DEAD** at every reachable depth: LMP −58 @d4, −55 @d6. Never
  revisit LMP/futility/razor (would need d10+, unreachable).
- **Eval-strip is Elo-negative**, **lazy_margin tapped** — the cheap d4 cyc levers
  are tapped. Eval is well-optimized (incremental mat/PST, O(1) lazy stand-pat).
- **Bigger TT win** (measured): TT8→TT16 = −25% nodes @d6 (knee TT14=192KB); the
  window/DMA access penalty is ~0.15% of cyc/move — NEGLIGIBLE. Net ~−25% cyc/move,
  grows with depth. **Extra RAM (REU/XRAM) IS a real lever, validated on REU.**

## Tooling / gotchas (hard-won)
- **Validate on REAL emulation — structural checks LIE.** This session: a banked
  build "passed" everything but was a non-bootable PRG (`.basic_header` displaced
  from $0801 by a custom `-Wl,-T`); and the REU/ultimate profiles drop a symbol the
  gate can't see. Always boot/run it.
- **VICE text monitor is flaky** for game-driving (broken pipe, reconnect-timeout,
  wedge — single connection only). FINE for one static peek (spin-loop + a result
  byte, like `reu_dma_test.c`). The project built the fast core to AVOID VICE.
- **mos-sim flat core** (`tools/llvmmos_bench/build_caissa.sh` + `/tmp/validate`) is
  the robust 6502 validator — but it's flat 64K, so it CAN'T model banking/REU/XRAM
  or a >64K TT. Recipe to validate a profile's engine: build mos-sim image +
  `cref_mos` both with the profile -D, `/tmp/validate img map fenlist depth oracle`.
  (Used it for the Ultimate: 30/30 @d4+d6.)
- Gate: `bash tools/llvmmos_bench/speed_gate.sh` (~5-30 min). cyc/move ~168.7M (50-FEN).
- Strength: `tools/native_vs_stockfish.py` — jobs≤cores, SEQUENTIAL (parallel SF
  starves → inflates native Elo).
- `cref_mos` goes stale — `make cref_mos` after any engine change.
- memcfg `CREF_TT_BITS` is per-profile (lines ~45 HOST / 59 ULTIMATE / 70 NOVA /
  77 C64) — `sed` the RIGHT line (line numbers shift when profiles are added).

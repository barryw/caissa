## Tooling / gotchas (hard-won)
- **`cc -E -dM -DCREF_PROFILE_X`** to confirm a non-default profile actually selected —
  the memcfg auto-default guard can silently shadow it (that was the whole "bug #2").
- **Validate on REAL emulation.** Isolated DMA round-trip tests + structural checks LIE
  (the REU IRQ bug passed every micro-test; only a full search caught it). For EGTB the
  analog is the **index-parity test** + the **forced-mate-in-DTM play-out**.
- **`validate.c` now checks node counts** (`[ND]` tag), not just moves — a move-only gate
  hides config/TT mismatches.
- **VICE headless** (`/tmp/x64sc_stable`): needs the FULL flag set incl. **`-keepmonopen`**
  or the monitor port never opens. Drive via `tools/vice_caissa.py` (set `CAISSA_REU=1`
  to attach `-reu`). One x64sc at a time (shared `~/.cache/vice` autostart) — don't run
  concurrent VICE jobs, and **don't `pkill x64sc` while another job uses it**.
- **`-keybuf` is flaky** for driving the interactive game (`\n` 0x0A vs C64 RETURN 0x0D
  + timing) — it garbles input. For move-correctness use the SERVER (poke/peek ABI) or a
  host path test, NOT keybuf.
- **Python stdout is block-buffered to a file** — long `reu_validate.py` runs show no
  output until done; use `python3 -u` and write to a file you can `tail`.
- EGTB gen gotchas: **distance-ordered BFS** (FIFO over-estimates DTM); **readiness
  guard** (an unloaded table must be inert, not a false draw); `kk_idx` came out **564**
  not 462 (no diagonal-degeneracy reduction — simpler, +22 %, fine).
- `make verify` needs `build/texel_data.json` (git-tracked but may be absent in a fresh
  worktree — copy from the main checkout).
